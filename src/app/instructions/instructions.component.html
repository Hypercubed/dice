Decode In Case of Emergency (DICE) is a simple online and open-source tool to convert confidential messages into encrypted QRCodes to be decoded in case of emergency (or for any reason whatsoever).

Messages are encoded using an encryption pass phase and aes-256-cbc encryption, the same as the one used in the OpenSSL cli tool (tested with OpenSSL v1.1). All encryption and decryption happens within the browser and never sent to any server. DICE is free and open source. Feel free to deploy your own site for this tool; messages can always be decrypted using the OpenSSL cli tool as well.

How to encrypt

1 Pass phase

This pass phase is kept secret or shared with your intended recipient(s). Use long passwords and keep it safe. This pass phase cannot be recovered.

2 Message

The text will be encrypted using the encryption pass phase you entered. The text is encrypted with aes-256-cbc encryption, the same as the one used in the OpenSSL cli tool.

3 Download

Copy or download the encrypted text and/or QR Code. These can be embedded in a document, printed and stored in case of emergency. The QR code data includes the encrypted text as well as a link to this site.

How to decrypt

1 Pass phase

This is your secret pass phase or one shared with you. Press OK when ready.

 2 Encrypted text

Scan the QR code or enter the encrypted text. If you arrived at this site by scanning a QRcode this will be populated already.

3 Decrypted text

The text will be decrypted using the encryption pass phase you entered.

FAQ

Why should I use DICE?

DICE is free and open source. No software or installation is required. You can encrypt any text quickly using your own secret pass phase. After the encryption is done, your message will be unreadable without the pass phase. The encryption algorithm is run in the browser. No data is sent or logged. All messages are encrypted using 256-bit AES.

Use this service when you want to...

  • Send your sensitive messages to your friends or family.
  • Safely encode sensitive information for storage online.
  • Safely backup sensitive information to paper for use by yourself or a trusted third party in case of emergency.

Where are the encrypted messages stored?

They are not stored by this site at all. If you would like to store the encrypted text or QR codes you will need to copy them to a document for storage or printing.

How safe is DICE?

All messages are encrypted using AES with a 256-bit key length, (trusted by governments and corporations all over the world) along with well known key derivation function PBKDF2. For the highest level of security use a strong pass phase. Feel free to deploy your own server for this tool or use the OpenSSL cli tool.

We do not store any data about your message or encryption pass phase.

We believe this service is safe; however all use is completely at your own risk.

 Can I retrieve a forgotten pass phase?

Nope. Pass phases and messages never leave your browser. There is no backup unless you made it yourself. If you've forgotten, it's gone for good.

Where can I find the source code?

All code is open-source on GitHub.

What if this site is shut down?

Messages are encrypted and encoded in the same format as OpenSSL cli tool.

Use OpenSSL to encode: echo -n "{{secret message}}" | openssl enc -e -aes-256-cbc -base64 -A -pbkdf2 -md sha256 -iter 10000

Use OpenSSL to decode: echo -n "{{encrypted message}}" | openssl enc -d -aes-256-cbc -a -A -pbkdf2 -md sha256 -iter 10000

Use OpenSSL and qrcode to generate a qr code: echo -n "{{secret message}}" | openssl enc -e -aes-256-cbc -a -A -pbkdf2 | qrcode
